# v1.0.0 Release Notes

`v1.0.0` is the first production-readiness milestone for the gateway-backed ManuscriptPrep stack.

## Highlights

- PostgreSQL-backed gateway persistence
- queued worker execution outside the gateway process
- health, readiness, queue, and worker visibility
- token authentication with ownership enforcement
- managed manuscript and configuration profile records
- artifact indexing with checksum metadata
- environment-driven compose deployment defaults
- non-root gateway and worker containers
- gateway-served operator dashboard at `/ui`

## Included Slices

- `P0`: asynchronous worker execution
- `P1`: operational health, readiness, and recovery
- `P2`: authentication and identity
- `P3`: manuscript and configuration management
- `P4`: artifact storage hardening
- `P5`: deployment and security hardening
- `P6`: web UI and operator experience

## Known Limits

- the worker model is still polling-based rather than queue-broker-based
- the operator web UI is intentionally minimal and API-driven
- artifact storage is still local-filesystem backed, with metadata centralized in the gateway store

## Suggested Next Milestones

- broker-backed worker dispatch
- richer user and manuscript lifecycle management
- object-storage backend for artifacts
- more complete web-based job creation and manuscript submission workflows
